## Part 1: Breathing Video System

### How It Works

1. **Infrastructure Ready**: HTML containers accept `data-video-src` attribute
2. **Graceful Fallback**: Shows static image if no video exists
3. **Lazy Loading**: Videos only play when scrolled into viewport
4. **Performance**: Pauses when out of view, optimized for mobile

### HTML Structure

```html
<div
  class="living-ink__breathing"
  data-video-src="/assets/videos/breathing/IMG_0159_breathing.mp4"
  data-fallback-src="/assets/drawings/IMG_0159.jpg"
  data-alt="Sadhu meditation breathing"
>
  <!-- Auto-generated by living-ink.js -->
</div>
```

### Generating AI Breathing Videos

#### Method 1: Replicate API (Recommended)

Use Stable Video Diffusion or AnimateDiff for subtle motion:

```javascript
// replicate-breathing-generator.js
import Replicate from 'replicate';

const replicate = new Replicate({
  auth: process.env.REPLICATE_API_TOKEN,
});

async function generateBreathingVideo(imagePath) {
  const output = await replicate.run(
    "stability-ai/stable-video-diffusion:3f0457e4619daac51203dedb472816fd4af51f3149fa7a9e0b5ffcf1b8172438",
    {
      input: {
        input_image: imagePath,
        cond_aug: 0.02,           // Low = subtle motion
        decoding_t: 7,            // Frames
        video_length: "14_frames_with_svd",
        sizing_strategy: "maintain_aspect_ratio",
        motion_bucket_id: 40,     // Low = gentle breathing
        frames_per_second: 6
      }
    }
  );

  return output; // URL to generated video
}

// Generate for all key drawings
const drawings = [
  '/assets/drawings/IMG_0159.jpg',  // Sadhu
  '/assets/drawings/IMG_0161.jpg',  // Shaolin
  '/assets/drawings/IMG_0165.jpg',  // Jimi Hendrix
];

for (const drawing of drawings) {
  console.log(`Generating breathing video for ${drawing}...`);
  const videoUrl = await generateBreathingVideo(drawing);
  console.log(`✓ Generated: ${videoUrl}`);
  // Download and save to /assets/videos/breathing/
}
```

**Settings for "Breathing" Effect:**
- `motion_bucket_id`: 20-40 (low = subtle)
- `cond_aug`: 0.01-0.03 (very low = stay close to original)
- `video_length`: 14 frames (short loop)
- `frames_per_second`: 6 (smooth but not hyper-realistic)

#### Method 2: RunwayML Gen-2

Alternative if you want more control:

```python
# runway_breathing.py
import requests

RUNWAY_API_KEY = 'your_key_here'

def generate_breathing(image_path):
    response = requests.post(
        'https://api.runwayml.com/v1/generate',
        headers={'Authorization': f'Bearer {RUNWAY_API_KEY}'},
        json={
            'model': 'gen2',
            'prompt': 'extremely subtle breathing motion, barely moving, serene',
            'image': image_path,
            'duration': 4,  # seconds
            'motion_amount': 1,  # 1-10 scale, use 1-2 for breathing
        }
    )
    return response.json()['output_url']
```

### Creating Custom Diagrams

Add to `SVG_PATHS` object in `living-ink.js`:

```javascript
const SVG_PATHS = {
  // ... existing paths ...

  customSpiral: `
    M 150 150
    Q 150 140, 160 140
    Q 170 140, 170 150
    Q 170 165, 155 165
    Q 135 165, 135 150
  `,

  brandLogo: `
    M 50 50
    L 250 50
    L 250 180
    L 50 180
    Z
    M 100 100 L 200 100
  `
};
```

**Path Tips:**
- Use relative commands (m, l, q, c) for easier editing
- Keep complexity moderate (20-50 commands max)
- Test path length: `const path = document.querySelector('path'); path.getTotalLength();`
- Longer paths = longer draw duration

## Part 4: Performance

### Mobile Optimization

Automatically applied:
- Animation duration reduced by 30% on mobile
- No hover effects on touch devices
- Videos pause when out of viewport
- Reduced motion if user prefers

### Bundle Size

- **living-ink.js**: ~8KB uncompressed, ~1.2KB gzipped
- **living-ink.css**: ~6KB uncompressed, ~1.8KB gzipped
- **Total**: ~3KB additional payload (minimal)

### Performance Tips

1. **Lazy load videos**: Use `loading="lazy"` attribute
2. **Limit simultaneous diagrams**: 3-4 per viewport max
3. **Compress videos**: Use H.264 with 3-5Mbps bitrate
4. **Shorter loops**: 2-4 seconds is ideal for breathing effect
5. **Test on mobile**: Ensure 60fps on mid-range devices

## Part 5: Accessibility

### Built-in Features

- **Prefers Reduced Motion**: Disables all animations
- **High Contrast Mode**: Increases stroke width
- **Screen Readers**: Proper `aria-label` and `alt` text
- **Keyboard Navigation**: All interactive elements focusable
- **Print Styles**: Shows static images only

### Best Practices

```html
<!-- Always include alt text -->
<div class="living-ink__breathing"
  data-fallback-src="/path/to/image.jpg"
  data-alt="Detailed description for screen readers"
>
</div>

<!-- Hide decorative diagrams -->
<div class="living-ink__diagram"
  data-diagram-type="bagua"
  aria-hidden="true"
>
</div>
```

## Part 9: Troubleshooting

### Videos Not Playing

1. Check video file path is correct
2. Verify video codec (H.264 is most compatible)
3. Ensure video is hosted on same domain (CORS)
4. Check browser console for errors
5. Test with `data-video-src` attribute set

### Diagrams Not Drawing

1. Verify `data-diagram-type` matches key in `SVG_PATHS`
2. Check ScrollTrigger is registered: `gsap.registerPlugin(ScrollTrigger)`
3. Ensure element is in viewport
4. Inspect if SVG was injected: `document.querySelector('.living-ink__svg')`

### Performance Issues

1. Reduce number of simultaneous animations
2. Lower video bitrate/resolution
3. Increase animation durations (smoother)
4. Use IntersectionObserver to pause off-screen videos
5. Check browser DevTools Performance tab
